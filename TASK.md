# 🧾 Product Spec: `gwatch`

## 1. Общее описание

`gwatch` — CLI-утилита (TUI), вдохновленная и построенная на исходном коде `gping`, позволяющая в реальном времени строить графики значений, получаемых из stdout пользовательских команд.

В отличие от `gping`, который визуализирует latency или время выполнения команд, `gwatch` визуализирует **произвольные числовые значения**, извлекаемые из результатов выполнения команд.

---

## 2. Цели продукта

### Основная цель

Дать пользователю простой способ:

* запускать команды периодически
* извлекать числовые значения
* визуализировать их как time-series в терминале

### Примеры use-case

* мониторинг CPU/Memory конкретного процесса
* наблюдение за метриками системы через shell
* быстрый ad-hoc мониторинг без Prometheus/Grafana
* отладка производительности

---

## 3. Основная модель

### Концепция

Каждый `watch` — это:

> источник данных → поток числовых значений → линия на графике

---

### Жизненный цикл одного `watch`

1. С заданной периодичностью выполняется команда (`exp`)
2. Читается stdout
3. Из stdout извлекается число
4. Если число получено → добавляется точка на график
5. Если нет → значение пропускается

---

## 4. Поведение (runtime)

### Выполнение

* Все `watch` выполняются **независимо и параллельно**
* Каждый имеет свой `period`
* Если `period` не указан → используется глобальный

---

### Обработка результата

#### Успешный случай

* берётся **первая строка stdout**
* парсится как `float`

#### Ошибки

Если:

* stdout пустой
* значение не число
* команда завершилась с ошибкой

👉 точка **не добавляется** (аналогично packet loss в gping)

---

## 5. Визуализация (TUI)

Наследуется от `gping`.

### Основные свойства

* один график
* несколько линий (по одной на `watch`)
* X — время (скользящее окно)
* Y — значение

---

### Поведение графика

* авто-масштаб по Y
* фиксированное окно по X (buffer, как в gping)
* обновление в реальном времени
* легенда с названиями (`title`)

---

### Отображение ошибок

* пропуски значений не соединяются линией (разрывы)

---

## 6. Конфигурация

Поддерживаются два способа:

---

# 6.1 Основной способ: TOML

### CLI

```bash
gwatch -f config.toml
gwatch -f -          # stdin
```

---

### Пример конфигурации

```toml
period = 1

[[watch]]
title = "Memory"
exp = "ps aux | grep oxide | grep -v grep | awk '{print $6}'"
measure = "bytes"

[[watch]]
title = "CPU"
exp = "ps aux | grep oxide | grep -v grep | awk '{print $3}'"
period = 1
measure = "float"
```

---

### Поля

#### Глобальные

| поле   | тип   | Дефолт  |описание            |
| ------ | ----- | ------- |------------------- |
| period | float |    1    |период по умолчанию |

---

#### `[[watch]]`

| поле    | тип    | обязательность |  дефолт       | описание                |
| ------- | ------ | -------------- | ------------- | ----------------------- |
| exp     | string | да             |               | shell-команда           |
| title   | string | нет            | Пустая строка | имя линии               |
| period  | float  | нет            |    1          | индивидуальный интервал |
| measure | string | нет            |   float       | тип значения            |

---

### measure

| значение | описание                      |
| -------- | ----------------------------- |
| float    | обычное число                 |
| bytes    | автоформатирование (KB/MB/GB) |

---

## 6.2 Альтернативный способ: CLI mini-DSL

### CLI

```bash
gwatch \
  --watch 'title=Memory;measure=bytes;period=2.5;exp=...' \
  --watch 'title=CPU;measure=float;period=1;exp=...'
```

---

### Формат

```
key=value;key=value;...
```

### Обязательные поля

* `exp`

---

### Особенности

* порядок полей не важен
* отсутствующие поля → дефолты
* разделитель `;`

---

## 7. CLI интерфейс

### Основные аргументы

| аргумент            | описание                 |
| ------------------- | ------------------------ |
| `-f, --file <path>` | путь к TOML              |
| `-f -`              | читать из stdin          |
| `--watch <spec>`    | добавить watch через DSL |
| `-p <seconds>`      | глобальный период        |

---

### Правила

* нельзя одновременно использовать `-f` и `--watch`
* если не указан ни один источник → ошибка

---

## 8. Наследуемые возможности gping

Используются без изменений:

* TUI рендеринг
* event loop
* буфер значений
* управление частотой обновления UI
* работа с несколькими источниками
* цвета линий

---

## 9. Расширения относительно gping

| Фича         | gping            | gwatch                       |
| ------------ | ---------------- | ---------------------------- |
| Источник     | ping / exec time | stdout                       |
| Интервал     | глобальный       | per-watch                    |
| Тип значения | latency          | типизированный (`measure`)   |
| Парсинг      | встроенный       | пользовательский (через exp) |

---

## 10. Обработка ошибок

### Ошибки конфигурации

* отсутствует `exp`
* неверный `period`
* неизвестный `measure`

→ ошибка при старте

---

### Ошибки выполнения

* команда завершилась с ошибкой
* stdout не число

→ точка пропускается

---

## 11. Нефункциональные требования

### Производительность

* минимальная нагрузка
* не блокировать UI

### Надёжность

* зависшая команда не должна блокировать остальные

(опционально в будущем: timeout)

---

## 12. Возможные будущие расширения

* regex / парсинг stdout

---
